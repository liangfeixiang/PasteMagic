🔐 RSA密钥生成不自动保存功能测试

📋 测试目标

原始需求：generateRSAKeys 只更新输入框的值，不saveConfig

修改内容：

  • 移除 generateRSAKeys 函数中的自动 saveConfig 调用
  • 添加 onUpdateCallback 参数用于更新输入框值
  • 修改 ConfigEditor 中的 handleGenerateKeys 调用方式
  • 添加用户友好的提示信息

❌ 修改前（自动保存）

const generateRSAKeys = async (configToUpdate) => { // ... 密钥生成逻辑 ... const updatedConfig = { ...configToUpdate, publicKey: publicKeyPEM, privateKey: privateKeyPEM }; saveConfig(updatedConfig); // ❌ 自动保存 toast.success('RSA密钥对生成成功！'); };

问题：生成密钥后立即保存，用户无法预览或修改

✅ 修改后（仅更新输入框）

const generateRSAKeys = async (configToUpdate, onUpdateCallback) => { // ... 密钥生成逻辑 ... const updatedConfig = { ...configToUpdate, publicKey: publicKeyPEM, privateKey: privateKeyPEM }; // ✅ 只更新输入框值，不自动保存 if (onUpdateCallback && typeof onUpdateCallback === 'function') { onUpdateCallback(updatedConfig); } toast.success('RSA密钥对生成成功！请手动点击保存按钮保存配置'); };

优势：用户可以预览生成的密钥后再决定是否保存

🔧 ConfigEditor 调用修改

修改前：

const handleGenerateKeys = () => { onGenerateKeys(editedConfig); // 直接调用 };

修改后：

const handleGenerateKeys = () => { onGenerateKeys(editedConfig, (updatedConfig) => { setEditedConfig(updatedConfig); // 回调更新本地状态 }); };

🎯 用户体验改进

📋 预览功能

生成密钥后可以在输入框中预览公钥和私钥内容

⚠️ 手动确认

需要用户主动点击保存按钮才会持久化配置

🔄 灵活修改

可以在保存前修改生成的密钥或其它配置项

🧪 功能验证测试

测试步骤：

  1. 打开秘钥配置管理界面
  2. 选择或创建一个RSA配置
  3. 点击"生成RSA密钥对"按钮
  4. 观察公钥和私钥输入框是否被填充
  5. 检查是否显示"请手动点击保存按钮"的提示
  6. 修改生成的密钥内容（可选）
  7. 点击保存按钮确认保存

预期结果：

  • ✅ 密钥生成成功，输入框被正确填充
  • ✅ 不会自动保存到存储中
  • ✅ 显示明确的操作提示
  • ✅ 用户可以预览和修改生成的密钥
  • ✅ 只有点击保存按钮后才持久化配置

📝 代码变更总结

主要修改点：

generateRSAKeys 函数：
  • 添加 onUpdateCallback 参数
  • 移除 saveConfig(updatedConfig) 调用
  • 修改成功提示信息
ConfigEditor 组件：
  • 修改 handleGenerateKeys 实现
  • 传入回调函数更新本地状态
  • 保持原有UI交互逻辑

💡 设计理念：

遵循"用户控制"原则，让用户能够预览和确认自动生成的内容，避免意外的数据变更。
